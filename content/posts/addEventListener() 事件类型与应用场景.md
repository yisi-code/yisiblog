## addEventListener() 事件类型与应用场景

### 鼠标事件

| 事件类型 | 描述 | 应用场景 |
|:---|:---|:---|
| `click` | 鼠标点击元素时触发 | 按钮提交、项目选择、交互触发 |
| `dblclick` | 鼠标双击元素时触发 | 编辑模式切换、快速执行操作 |
| `mousedown` | 鼠标按键在元素上被按下时触发 | 拖拽开始、自定义按下效果 |
| `mouseup` | 鼠标按键在元素上被释放时触发 | 拖拽结束、自定义按钮反馈 |
| `mousemove` | 鼠标在元素内部移动时触发 | 鼠标跟踪、拖拽过程中的元素移动 |
| `mouseenter` | 鼠标指针移入元素时触发（不冒泡） | 下拉菜单显示、悬停提示 |
| `mouseleave` | 鼠标指针移出元素时触发（不冒泡） | 下拉菜单隐藏、悬停效果结束 |
| `mouseover` | 鼠标指针移入元素或其子元素时触发（会冒泡） | 需要感知子元素进入的复杂交互 |
| `mouseout` | 鼠标指针移出元素或其子元素时触发（会冒泡） | 需要感知子元素离开的复杂交互 |
| `contextmenu` | 用户右击元素时触发 | 自定义右键菜单 |
| `wheel` | 鼠标滚轮在元素上下滚动时触发 | 页面缩放、画布滚动 |

### 键盘事件

| 事件类型 | 描述 | 应用场景 |
|:---|:---|:---|
| `keydown` | 键盘按键被按下时触发 | 游戏控制、快捷键、实时验证 |
| `keyup` | 键盘按键被释放时触发 | 输入完成确认、停止连续动作 |
| `keypress` | 按键被按下并松开时触发（注意：已逐渐被弃用） | 字符输入（历史用法） |

### 表单事件

| 事件类型 | 描述 | 应用场景 |
|:---|:---|:---|
| `submit` | 表单被提交时触发 | 表单数据最终验证与提交 |
| `input` | 表单元素的值发生变化时实时触发 | 实时搜索、字符计数、输入提示 |
| `change` | 表单元素的值发生变化且失去焦点后触发 | 下拉选择确认、文件选择 |
| `focus` | 元素获得焦点时触发 | 输入框高亮、显示提示 |
| `blur` | 元素失去焦点时触发 | 表单验证、隐藏提示 |
| `focusin` | 元素即将获取焦点时触发 | 更早的焦点进入处理 |
| `focusout` | 元素即将失去焦点时触发 | 更早的焦点离开处理 |
| `reset` | 表单重置时触发 | 表单清空后的后续操作 |

### 窗口/文档/资源事件

| 事件类型 | 描述 | 应用场景 |
|:---|:---|:---|
| `load` | 页面及所有依赖资源（如图片）完成加载时触发 | 初始化操作、显示页面内容 |
| `DOMContentLoaded` | HTML 文档完全加载和解析完成后触发（不等待样式表、图片） | 早期 DOM 操作，比 `load` 更快 |
| `resize` | 浏览器窗口大小改变时触发 | 响应式布局调整、图表重绘 |
| `scroll` | 元素或页面滚动时触发 | 无限滚动、导航栏显隐、懒加载 |
| `beforeunload` | 用户试图关闭页面或导航离开前触发 | 提示保存未提交的数据 |

### 拖拽事件

| 事件类型 | 描述 | 应用场景 |
|:---|:---|:---|
| `dragstart` | 用户开始拖拽元素时触发 | 设置拖拽数据 |
| `drag` | 在拖拽过程中持续触发 | 更新拖拽视觉反馈 |
| `dragover` | 被拖拽元素在有效放置目标上方移动时触发 | 确定有效的放置目标 |
| `drop` | 被拖拽的元素被放置在目标区域时触发 | 处理拖放的数据交换 |
| `dragend` | 拖拽操作结束时触发 | 清理拖拽状态 |

### 剪贴板事件

| 事件类型 | 描述 | 应用场景 |
|:---|:---|:---|
| `copy` | 用户拷贝元素内容时触发 | 自定义复制内容格式 |
| `cut` | 用户剪切元素内容时触发 | 自定义剪切行为 |
| `paste` | 用户粘贴内容时触发 | 验证或清理粘贴内容 |

### 多媒体事件

| 事件类型 | 描述 | 应用场景 |
|:---|:---|:---|
| `play` | 视频/音频开始播放时触发 | 控制界面更新 |
| `pause` | 视频/音频暂停时触发 | 控制界面更新 |
| `ended` | 视频/音频播放结束时触发 | 自动播放下一曲/集 |

### CSS 动画与过渡事件

| 事件类型 | 描述 | 应用场景 |
|:---|:---|:---|
| `animationstart` | CSS 动画开始播放时触发 | 动画开始时的联动操作 |
| `animationend` | CSS 动画结束播放时触发 | 动画结束后的清理或链式动画 |
| `transitionend` | CSS 完成过渡后触发 | 过渡效果完成后的回调 |

## 以Mouseout 事件机制详解阐述事件冒泡

### 现象描述

当使用 `mouseout` 事件监听时，会出现以下触发序列：

1. **鼠标从父元素移入第一个子元素** 

   - 父元素会触发一次 `mouseout` 事件

   - 第一个子元素会触发 `mouseover` 事件（该事件会冒泡到父元素）

2. **鼠标从第一个子元素移出，但仍处于父元素内** 

   - 第一个子元素会触发 `mouseout` 事件（即使子元素没有绑定监听）

   - 父元素会再次触发 `mouseout` 事件（由于事件冒泡）

### 核心原理

#### 事件触发机制

- **`mouseout` 触发条件** ：当鼠标指针离开一个元素时触发，包括移动到其子元素

- **事件冒泡** ：子元素触发的事件会沿 DOM 树向上传播，触发父元素的同名监听器

#### 关键技术概念

```javascript
// 事件对象包含的关键信息
event.target; // 实际触发事件的元素
event.currentTarget; // 绑定事件监听器的元素
event.relatedTarget; // 鼠标移动到的目标元素
```

#### 若要只针对某一目标，可使用不冒泡事件，如 mouseenter/mouseleave

```javascript
// 更简单可靠的方案
parentElement.addEventListener('mouseleave', function(e) {
// 只有当鼠标完全离开父元素及其所有子元素时触发
console.log('鼠标真正离开了父元素区域');
});
```

**优势** ：

- 不冒泡，只在绑定元素上触发

- 基于整个元素区域（包含所有后代）判断

- 避免内部子元素移动造成的误触发

#### 进阶方案：精确控制 mouseout

```javascript
// 如果需要使用 mouseout，可通过 relatedTarget 过滤
parentElement.addEventListener('mouseout', function(e) {
// 只有当鼠标完全移出父元素区域时才执行逻辑
if (!this.contains(e.relatedTarget)) {
console.log('鼠标真正离开了父元素区域');
}
});
```

### 实际应用建议

对于需要检测鼠标是否在整体区域内的场景， **强烈推荐使用 `mouseenter` / `mouseleave` 等不冒泡事件对** ，因为：

- 逻辑更清晰直观

- 避免复杂的冒泡处理

- 代码更易维护和调试
